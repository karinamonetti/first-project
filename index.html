Información útil antes de empezar

  1. HTML estructura el contenido del sitio a través de etiquetas; estas etiquetas son contenedores o "cajas". Estas cajas, también conocidas como elementos padres, pueden contener otras cajas dentro, siendo éstos los elementos "hijos". Para reconocer cuándo es un elemento padre y cuándo es un elemento hijo, debemos fijarnos en la anidación en el código, o sea, si un elemento contiene dentro otros, los elementos hijos podrán ser identificados por una tabulación que los precede.
  2. No todas las etiquetas tienen cierres (<></>) sino que hay algunas que son etiquetas de apertura y de cierra al mismo tiempo (</>)
  3. Un comentario es una sección del código que el navegador no lee, solo sirve para quién tenga acceso al código y para brindar información sobre lo que representa el mismo. Los comentarios en HTML se crean con los siguientes símbolos: <!-- Esto es un comentario--> En VSCode el atajo por default para escribirlos es CONTROL+K CONTROL+C
  4. Mientras uno va aprendiendo es interesante analizar cómo están estructurados los sitios, cómo están maquetados y cómo se les ha aplicado estilos. Esto se puede hacer, desde el navegador y dentro del sitio web apretando F12 o yendo a las herramientas del navegador y buscando la opción "Herramientas para desarrolladores".

Introducción

HTML es un lenguaje de marcado, sus siglas en inglés significan HyperText Markup Language que en español equivale a "Lenguaje de Marcado de Hipertexto" y sirve para estructurar un sitio web. Para poner las cosas más claras sobre las diferencias entre HTML, CSS y JavaScript: HTML sirve para estructurar un sitio, por ejemplo, para indicarle al sitio que un párrafo es un párrafo, que un título es un título, etc., vendría a ser el "esqueleto" del sitio. CSS le da colores, cambia formas y posiciones a esa estructuración lograda por HTML y JavaScript se encarga de la interactividad del sitio, por ejemplo, eventos que ocurren al darle click a un botón.
